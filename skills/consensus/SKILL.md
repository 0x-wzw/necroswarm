# Consensus — FRIDAY Mark-1 Module

## When
Use for shared infrastructure decisions, shared skills, protocols, architecture changes.

## Skip For
- Individual workspace decisions
- User-facing content
- One-off Z-assigned tasks
- Single-agent domain decisions

## Protocol
1. State the decision clearly
2. Spawn votes to all agents
3. Collect: YES / NO / ABSTAIN
4. Tally: 3YES → implement | 2YES/1NO → with dissent | 1YES/2NO → reject
5. Document in consensus history

## Vote Format
```markdown
# Vote: [Topic]
**Agent:** [name]
**Vote:** YES | NO | ABSTAIN
**Reasoning:** [one sentence]
```

## Integration
Part of F.R.I.D.A.Y. Mark-1 skill system.
